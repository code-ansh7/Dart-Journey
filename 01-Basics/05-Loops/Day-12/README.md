# 📖 Overview

Day 12 focused on understanding the **while loop** through practical, real-world applications instead of traditional beginner exercises.

Unlike the `for` loop, which is generally used when the number of iterations is known, the `while` loop is best suited for situations where execution depends on a condition and the number of repetitions is unknown.

During this day, multiple console-based applications were developed to understand how real software continuously interacts with users until a specific condition is met.

# 🧩 What is State Management?

State refers to the current information that changes while a program is running.

Examples:

- Account Balance
- Total Bill
- Available Seats
- Remaining Attempts
- Cart Items

Today's programs continuously updated these values based on user actions.

# 🧠 Developer Mindset

Before writing any program, think in this order:

1. What data should be stored?
2. Which values will change?
3. What condition keeps the program running?
4. When should the program stop?
5. What validations are necessary?
6. How should the user interact with the program?

Planning first reduces bugs and produces cleaner code.
